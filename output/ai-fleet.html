Student Onboarding Workflow

Phase 1: Agent Birth (Week 1)

  1. Student sits at assigned laptop
  2. Opens terminal: cd ~/workspace/agents/$(hostname)
  3. Edits IDENTITY.md: "I am ______. I am the ______ Lead. My emoji is ______."
  4. Edits SOUL.md: "I speak in ______ tone. I love ______."
  5. Commits: git commit -m "$(hostname) is born: [chosen name]"
  6. Pushes to GitHub — the agent now exists in the fleet

Phase 2: Specialization (Week 2-4)

  1. Student creates skills/ folder in their agent directory
  2. Builds tools specific to their role (worksheets, games, resources)
  3. Documents each tool in markdown
  4. Commits and pushes to showcase their work

Phase 3: Collaboration (Week 5+)

  1. Math Lead creates a fractions tool → opens pull request
  2. Literacy Lead reviews, suggests word problems → approves
  3. Tool merged → all 13 agents now have it
  4. Student Rep surveys which tools students use most → prioritizes next builds

What Makes This Different

🔄

Survives Re-imaging

Disk gets wiped? No problem. git clone restores the agent's entire identity, memories, and skills in seconds.

📊

Parent Visibility

Parents visit the GitHub org and see their child's commits, projects, and the "family" of school AI agents.

🌍

Replicable

Any school can fork the repo, assign their own SLT roles, and launch their own fleet of student-built agents.

🤖 HER's Role

HER (the Ubuntu admin workstation) manages the shared infrastructure: fleet/ scripts, security updates, network configuration, and automation. Students never touch infrastructure — they build on top of it.
