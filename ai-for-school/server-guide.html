Executive Summary

We propose building a school-owned AI server that brings artificial intelligence directly into the classroom — no VPNs, no paid subscriptions, no data leaving the school network. Every teacher and student gets instant access to lesson planning assistants, text simplification tools, vocabulary builders, reading assessments, and educational games. All running on hardware the school owns and controls.

💡 The Big Picture

One server. Zero subscriptions. Unlimited users. A single machine running open-source AI models can serve an entire school. Teachers save hours on lesson planning. Students get personalized learning support. The school stays fully compliant with data protection laws — because no student data ever leaves the building.

Why This Matters for Your School

The Problem Teachers Face

  • Blocked access — AI tools like ChatGPT, Gemini, and Claude are often blocked by school firewalls
  • Data privacy — Teachers cannot safely enter student information, assessment data, or internal curriculum into public AI services
  • Cost barriers — Premium AI subscriptions are expensive; teachers don't know where to start
  • Digital divide — Not all staff have equal technical skills or confidence with AI

The Solution: A School-Owned AI Server

  • Internal access only — Works on the school LAN. No internet required after setup
  • Data never leaves — All processing happens inside the school. Fully compliant with GDPR, FERPA, or your local data protection laws
  • Open-source — Zero licensing fees. Uses free tools: Ubuntu Linux, Ollama, and open-weight AI models
  • Educational — Designed specifically for teaching, not general chat

If budget is limited, start with the same rackmount chassis but reduce RAM to 128GB (4x 32GB used DDR4 ECC = ¥1,200) and use 1x Tesla A2 (16GB VRAM = ¥2,000). Total: ~¥8,850. You keep the full rackmount, redundancy, and 10GbE — just with less RAM and one GPU. Upgrade later by adding RAM sticks and a second GPU.

Why a Rackmount Server?

For a school environment with a server room, the rackmount server is the best long-term investment:

  • Enterprise form factor — 2U rackmount sits in the server room, not on a desk. Professional, secure, out of the way.
  • Redundant power — dual 2200W hot-swap PSUs. If one fails, the other keeps running. Zero downtime for students.
  • 10GbE networking — dual SFP+ ports onboard. Future-proof when the school upgrades network infrastructure. 10x faster than 1GbE for large AI model transfers.
  • Maximum expansion — 10 PCIe slots (can add 2–4 more Tesla A2s later), 8 hot-swap drive bays (add storage without opening the case).
  • Enterprise reliability — built for 24/7 datacenter operation.
  • 32 cores is enough — AI inference is GPU-bound, not CPU-bound. The Tesla A2s will be the bottleneck long before 32 EPYC cores max out.

When a Rackmount Isn't Possible

If your school has no server room or rack cabinet, a full-tower dual-CPU workstation is an alternative. Expect to pay roughly ¥13,650 (~$1,900 USD) for 64 cores, 256GB RAM, and 2x Tesla A2 — but you lose rackmount, redundant PSU, and 10GbE networking. It sits on the floor or a desk like a very large PC.

What IT Needs to Provide

To make this operational, the school's IT team or network administrator needs to allocate:

  1. Static IP Address — Reserve one IP on the school LAN so the server is always reachable at the same address (e.g., 10.x.x.x or 192.168.x.x)
  2. DNS Entry — Map a subdomain like ai.yourschool.edu or ai.yourschool.cn to the static IP so teachers and students access it by name, not by numbers
  3. Firewall Rule — Allow internal LAN traffic to the server (typically ports 80 and 443). No external internet exposure required
  4. Rack Space — One 2U slot in the server cabinet, plus power outlets. If no server room is available, a full-tower alternative can sit in an office or classroom
